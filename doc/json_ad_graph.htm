Json Format for A Graph Representing an ADFun object

Under Construction
This section is under construction and subject to change without backward compatibility

Format
See Json data types and syntax .

Token

White Space
Any sequence of white space, except within a string, terminates the current token and is otherwise ignored.

Non-Negative Integer
A non-negative integer is a non-empty sequence of the following characters: 0123456789+-eE..

Floating Point Number
A floating point number is a non-empty sequence of the following characters: 0123456789+-eE.. Note that there can't be any white space between a leading plus or minus sign and the rest of the number.

String
A string starts with the double quote character " and includes all the characters until the next double quote. The value of a string is the sequence of characters between the double quotes. Currently there is no support using the double quote as part of the value of a string.

Single Character
The following is a list of the single character tokens:
Token Usage
, separates entries in a list
: separates name from corresponding value
{ starts a list name : value pairs
} ends a list of name : value pairs
[ starts a list of values
] ends a list of values

operator
The Json format for an operator is
[   code,
    n_result,
    n_arg, [ first_arg, .., last_arg ],
   "name"
]
code
Each operator has a corresponding operator_enum value defined by 
namespace CppAD { namespace local { namespace json {
    enum operator_enum  {
        add_operator = 0,   // has 2 arguments and 1 result
        mul_operator = 1,   // has 2 arguments and  1 result
        n_operator   = 2    // number of operators defined so far
    };
} } }
Note that n_operator is the number of operators implemented so far, not a valid operator value. The value code above is the non-negative integer corresponding to the operator_enum value. These values are subject to change and programs that uses these values should easily adapt to this change; e.g., it could use the enum type instead of the corresponding integer values.

n_results
This is the number of results for the operator; i.e., the dimension of its range space.

n_arg
This is the number of argument values for the operator; i.e., the dimension of its domain space.

Arguments
The values first_arg , ..., last_arg , are the node indices corresponding to each of the arguments for this operator. They specify which previous result (result before this operator in the graph) correspond to each of the arguments to this operator.

name
Each operator also has a name which is the string corresponding to its enum symbol up to but not including the _op at the end. For example add is the name for the operator add_op. The name is used to make the Jason easier for humans to read and for checking that the enum value code is correct.

dynamic_ind_vec
This is the independent dynamic parameter vector; see dynamic .. The function can depend on these parameters, but no derivatives are computed with respect to these parameters. We use the non-negative integer n_dynamic_ind for the number of elements in this vector.

Node Indices
We use the notation start_dynamic = 1 . The node indices corresponding to the dynamic parameter vector are
    start_dynamic, ..., start_dynamic + n_dynamic_ind - 1
independent_vec
This is the independent variable vector. The function can depend on these variable and derivatives can be computed with respect to these variables. We use the non-negative integer n_independent for the number of element in this vector.

Node Indices
We use the notation
    start_independent = start_dynamic + n_dynamic_ind
 . The node indices corresponding to the independent variable vector are
    start_independent, ...,  start_independent + n_independent - 1
string_vec
This is the string vector which cannot change. These strings can be used to display information about the function. The Json format for string_vec is
    [ n_string, [ first_string, ..., last_string ] ]
 where the non-negative integer n_string is the number of elements in this vector and first_string , ..., last_string , are strings .

Node Indices
We use the notation
    start_string = start_independent + n_independent
 The node indices corresponding to the string vector are
    start_string, ...,  start_string + n_string - 1
constant_vec
This is the constant parameter vector. These parameters can be used to define the function and cannot change. The Json format for constant_vec is
    [ n_constant, [ first_constant, ..., last_constant ] ]
 where the non-negative integer n_constant is the number of elements in this vector and first_constant , ..., last_constant are floating point numbers .

Node Indices
We use the notation
    start_constant = start_string + n_string
 . The node indices corresponding to the constant vector are
    start_constant, ...,  start_constant + n_constant - 1
operator_vec
The Jason format for an operator_vec is
    [ n_operator, [ first_operator, ..., last_operator ] ]
 where the non-negative integer n_operator is the number of operators in this vector and first_operator , ..., last_operator is an operator .

Node Indices
For i = 1 , ..., n_operator , we use n_result[i] to denote the n_result value corresponding to the i-th operator. We define
    start_operator[1] = start_constant + n_constant
 and for i = 2 , ..., n_operator ,
    start_operator[i] = start_operator[i-1] + n_result[i-1]
 For i = 1 , ..., n_operator , the nodes corresponding to the i-th operator are
    start_operator[i], ..., start_operator[i] + n_result[i] - 1
dependent_vec
This is the vector of dependent variable node indices. This identifies which nodes in the graph correspond to dependent variables. The Json format for dependent_vec is
    [ n_dependent, [ first_dependent, ..., last_dependent ] ]
 where the non-negative integer n_dependent is the number of elements in this vector and first_dependent , ..., last_dependent are the node indices for the corresponding dependent variables.

AD Graph
The Json format for an AD graph is
{
    "n_dynamic_ind": n_dynamic_ind,
    "n_independent": n_independent,
    "string_vec":    string_vec,
    "constant_vec":  constant_vec,
    "operator_vec":  operator_vec,
    "dependent_vec": dependent_vec
}
